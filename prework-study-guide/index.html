Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • HTML are statements.
  • Remember to source things with src = ./folder name/item name.
  • The alt abbreviation describes the source you linked.
  • To create a link use a href = "link name" within an anchor.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Use the box model (W,M,B,D,C) to organize the elements you're affecting within CSS.
  • Don't forget to link the CSS file into your HTML code in order to activate its functionality.
  • Apply CSS to a specific paragraph, use the section class = the item from the CSS file followed by id ="name of section".
  • .(name of item) is then broken down into the property of the named item.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • Use git commands for push/pull requisitions as well as staging your items.
  • Always be sure to use 'git pull origin main' beforehand to make sure your branch is up to date.
  • git checkout is the equivalent of cd in the CLI.
JavaScript
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Always end each statement with a semicolon.
  • Always link your script on the HTML file between the footer and body at the bottom.
  • Use arrays to have a group of items under one title name.
  • Arrays are numbered 0, 1, 2, x+1.
  • Functions end with (). Remember to identify function variables first.
